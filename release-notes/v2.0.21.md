## What's changed

- Fixed Connect dropping the user's Wi-Fi association or wired default route entirely on Realtek, Killer, USB-Ethernet, and some Wi-Fi drivers. SwiftTunnel no longer issues any `Set-NetAdapterAdvancedProperty` calls during connect; the LSO-disable that ran in the background was the last remaining trigger for the NIC reset that was dropping the default route 4 seconds into the session.
- Sample-logged super-segment truncations in the tunnel forwarding path so any LSO-related correctness issue on unusual TCP traffic is observable in stlog instead of silent. For Roblox-style sessions (UDP game traffic, small TCP API calls) this is expected to stay at zero.
- Existing TSO/LSO marker files from older builds are still restored automatically on the next app launch, so users upgrading from 2.0.20 or earlier do not end up with adapter offload stuck off.
