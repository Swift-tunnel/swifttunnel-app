## What's changed

- Fixed the Connect button briefly returning to the idle "Connect" state while a connection attempt was still running in the background.
- The UI now ignores stale disconnected `vpn_get_state` snapshots during an active connect attempt, matching the existing protection for stale backend state-change events.
- Added a regression test for the stale polling case.
