## What's changed

- Replaced the IPv6 adapter-binding disable path with a `netsh advfirewall` block rule so connecting no longer causes Ethernet adapters to relink or temporarily lose the default route.
- Scoped the IPv6 block to public IPv6/NAT64 destinations and the selected adapter's firewall interface type, leaving local IPv6 loopback, link-local, multicast, and ULA traffic alone while connected.
- Crash/disconnect recovery now verifies that the IPv6 firewall rule was actually removed before deleting the recovery marker, so startup recovery can retry if cleanup fails.
- Legacy IPv6 binding markers still restore through the old `Enable-NetAdapterBinding` path for users upgrading while connected.
