## What's changed

- Added an in-app What's New update popup with release notes and an Update button, so users can update from SwiftTunnel without checking Discord or Settings.
- Expanded the main app layout so pages use the available window width better instead of sitting in a narrow centered column.
- Improved startup self-repair. SwiftTunnel now checks for stale driver/tunnel/network state on launch, repairs what it can, relaunches as admin when needed, and only asks for a Windows restart when the driver truly needs one.
- Fixed a stuck network state where Windows could keep SwiftTunnel's packet-filter binding active after a crash, failed session, forced close, or uninstall, making Wi-Fi/internet feel like it only worked when SwiftTunnel was running.
- Improved Tunnel Cleanup, Internet Recovery, disconnect cleanup, and uninstall cleanup so they clear stale WinpkFilter / nt_ndisrd bindings, old tunnel markers, hosts entries, IPv6 recovery markers, and WFP cleanup state.
- Made split-tunnel driver repair stricter. Repair now verifies that the driver is actually ready with TCP/IP-bound adapters instead of only checking whether the Windows driver handle can open.
- Added connect-time binding recovery so SwiftTunnel can re-enable a disabled WinpkFilter binding on the selected adapter before connecting, instead of telling users to run Repair forever.
- Fixed Route Assist texture/menu/chat/avatar issues by keeping Roblox assets, settings, auth, avatar, menu, and chat traffic direct unless Full Country Ban is enabled.
- Fixed Partial Bypass overlap with Route Assist. Partial Bypass now wins when both are on, keeps gameplay and assets direct for original ping, and only relays the Roblox join/control path for specific-game bans.
- Improved Full Country Ban for fully blocked countries. Full bypass now keeps Roblox settings, auth, assets, and gameplay on the relay instead of letting blocked direct paths leak back in.
