## What's changed

- Fixed the split-tunnel adapter attach regression that affected some PCs after v2.5.13.

- Improved WinpkFilter validation so stale Windows binding data no longer blocks adapters the driver already sees.

- Improved Manual adapter selection by blocking unusable adapters before starting the tunnel.

- Improved region selector.

- Improved Windows uninstall cleanup so deleting SwiftTunnel from Settings/Control Panel removes the WinpkFilter network component instead of leaving a broken adapter binding behind.

- Restored Partial Bypass asset routing so maps, textures, sounds, avatars, and accessories can use the relay while gameplay UDP stays direct.

- Improved Partial Bypass loading behavior for users whose direct Roblox asset path is blocked or unstable.

- Kept Full Country Ban as the full relay fallback for fully blocked countries.

- Fixed Full Country Ban closing Roblox after connecting. SwiftTunnel now asks users to close Roblox first, then connect and reopen it through the bypass.

- Added a Full Country Ban safety prompt when Roblox is already running, with an option for SwiftTunnel to close Roblox before enabling the bypass.

- Improved Full Country Ban login and verification routing for blocked countries.

- Improved Full Country Ban asset and Roblox service routing for blocked countries.
