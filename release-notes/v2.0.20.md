## What's changed

- Fixed Connect briefly killing the user's internet on some Ethernet adapters (Realtek, Killer, USB-Ethernet). The split-tunnel startup script now only disables Large Send Offload; it no longer toggles four separate checksum-offload registry keys, which were each triggering a transient NIC reset and dropping the IPv4 default route mid-connect.
- Added a 30-second post-connect adapter watchdog: if the OS reports no internet AND the relay isn't returning traffic for ~3s during the connect window, SwiftTunnel rolls back the session with a clear "please try again" message instead of leaving the user stuck waiting for the relay-dead timeout.
- Distinguished local-internet loss from relay loss in mid-session error messages so users on a flaky Wi-Fi or sleep/resume aren't told to "choose another relay" when the actual problem is on their end.
