## What's changed

- Fixed failed connects when relay policy lookup is temporarily unavailable. SwiftTunnel can now continue with a marked legacy fallback while still blocking fallback when the relay explicitly requires authentication.
- Prevented slow Windows adapter checks from stopping connect after 20 seconds. The split tunnel preflight now gets the same connection budget as the rest of connect.
- Reduced reconnect loops caused by transient default-route lookup failures. SwiftTunnel now preserves the known-good adapter binding until Windows reports a real route owner.

**Release**: https://github.com/Swift-tunnel/swifttunnel-app/releases/tag/v2.0.12
