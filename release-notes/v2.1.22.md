## What's changed

- Fixed Repair driver reporting success when the WinpkFilter binding was still missing on the active network adapter.
- Added a post-repair split tunnel binding check so SwiftTunnel now asks for a reboot when Windows has not restored the adapter binding yet.
- Improved repair safeguards so unrelated adapter detection issues do not get mistaken for the WinpkFilter binding repair path.
