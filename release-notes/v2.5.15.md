## What's changed

- Fixed the split-tunnel driver loop where Windows reported no TCP/IP-bound adapters and SwiftTunnel kept asking users to restart or repair manually.

- Improved automatic recovery when Windows blocks SwiftTunnel's network filter even though the app is already running as admin.

- Moved Adapter selection to the Connect screen and added clear usable/not-usable markers for manual adapter choices.

- Improved login reliability on networks where `auth.swifttunnel.net` is blocked or broken by DNS, proxy, antivirus, or ISP routing.

- Added a desktop password sign-in fallback through the SwiftTunnel website API without storing passwords or adding private keys to the app.

- Kept true driver-install reboot cases separate so SwiftTunnel does not spam repair loops when Windows really needs a restart.

- Updated tests for driver repair, elevated driver access recovery, and login fallback behavior.

Release: https://github.com/Swift-tunnel/swifttunnel-app/releases/tag/v2.5.15
