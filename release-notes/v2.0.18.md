## What's changed

- Fixed split tunnel setup getting stuck or falling back to Connect with "Driver initialization timed out" on machines where PowerShell or WMI is slow to respond.
- The IPv6 leak-prevention firewall rule is now installed and removed by launching `netsh.exe` directly instead of wrapping `netsh` inside PowerShell.
- Firewall-rule crash recovery and best-effort disconnect cleanup also use direct `netsh.exe`, so recovery no longer depends on PowerShell for the current IPv6 block method.
- `netsh.exe` is resolved from System32 like the other Windows system tools SwiftTunnel launches.

**Release**: https://github.com/Swift-tunnel/swifttunnel-app/releases/tag/v2.0.18
