## What's changed

- Fixed connections started after Roblox is already running. SwiftTunnel now seeds already-running matching game processes into the split tunnel watcher instead of waiting only for new process-start events.
- Improved bad-relay recovery when relay pings still answer but inbound game packets cannot be injected locally. SwiftTunnel now marks sustained data-plane injection failures as stale or dead so the session can recover instead of staying falsely healthy.
- Stopped treating the generic Microsoft Store app host as a Roblox tunnel signal, avoiding accidental tunneling decisions for unrelated Store apps.

**Release**: https://github.com/Swift-tunnel/swifttunnel-app/releases/tag/v2.0.13
