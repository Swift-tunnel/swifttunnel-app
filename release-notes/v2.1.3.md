# SwiftTunnel v2.1.3

This follow-up release completes the remaining networking review items from the v2.1.2 hardening pass.

- Enforces game-server lookup generations in the live auto-routing resolver path so stale resolver responses cannot switch relays after a newer server IP is observed.
- Reduces split-tunnel hot-path bookkeeping overhead for detected Roblox server IPs and removes the unused metric-blind `routes.rs` module.
- Tightens relay health telemetry by ignoring duplicate PONG sequence numbers instead of letting duplicate pongs hide loss.
- Improves WinpkFilter ARM64 driver selection by using structured Windows machine attributes before falling back to environment variables.

**Release**: https://github.com/Swift-tunnel/swifttunnel-app/releases/tag/v2.1.3
