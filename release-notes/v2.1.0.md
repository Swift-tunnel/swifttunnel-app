## What's changed

- Fixed fatal VPN networking failures that could leave SwiftTunnel showing Connected after the relay sender or split-tunnel worker had already failed.
- Improved recovery messages so driver-handle failures still guide users to reinstall the split-tunnel driver, while internal worker failures ask for a clean reconnect instead.
- Refined UltraBoost to keep only FPS-focused Roblox FFlags and remove the retired high-DPI sharpness override from existing client settings.
- Expanded the Windows split-tunnel testbench with HTTPS API probe coverage and safer probe-process cleanup.

**Release**: https://github.com/Swift-tunnel/swifttunnel-app/releases/tag/v2.1.0
