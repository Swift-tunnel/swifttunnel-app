## What's changed

- Fixed a split-tunnel failure path that could leave the network adapter in an intercepted state after a failed or timed-out VPN connect attempt. Failed connects now unwind the VPN path before the app offers retry, preventing the Wi-Fi-broken-until-reboot state reported by users.
- Added a backend connect timeout and cleanup fallback so split tunneling cannot sit indefinitely in a half-connected state when the driver or relay setup stalls.
- Tightened split-tunnel recovery so the binding repair retry only runs for the matching driver binding error and does not retry unrelated timeout failures.
- Improved Roblox boost cleanup behavior so missing or malformed Roblox settings are reported as warnings when the rest of the boost can still apply, while legacy QoS cleanup now targets only the exact old policy.
