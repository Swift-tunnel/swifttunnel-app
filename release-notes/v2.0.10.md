## What's changed

- Fixed adapter binding setup on systems where Windows exposes a different interface alias than the hardware description. SwiftTunnel now uses the operational alias, such as `Ethernet 2`, for IPv6 and adapter binding changes instead of the Realtek hardware description that PowerShell cannot target with `-Name`.
- Prevented false connected states when relay authentication succeeds but the relay never returns tunneled traffic. Dead relay health now transitions the session to an error and cleans up the partial tunnel.
- Made IPv6 hardening failures block connect instead of allowing an IPv4-only tunnel to report success while IPv6 traffic may bypass it.
